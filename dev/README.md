# Implementation Notes

This directory contains implementation details, patterns used, and known issues.

## Purpose
Document the **"how"** it was actually built - implementation patterns, code organization, and practical details.

## When to Add Implementation Notes
- Documenting implementation patterns
- Recording known issues and workarounds
- Sharing lessons learned
- Noting technical debt
- Explaining performance optimizations

## Format
See `.cursorrules` in this directory for the template and guidelines.

## Examples
- `queue-system-implementation.md`
- `vehicle-valuation-service-notes.md`
- `mot-reminder-implementation.md`
