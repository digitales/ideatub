# Architectural Decisions

This directory contains architectural decision records (ADRs) and design choices.

## Purpose
Document the **"why"** behind technical choices, architectural patterns, and strategic decisions.

## When to Add a Decision
- Choosing between technologies or frameworks
- Defining system architecture patterns
- Making trade-offs between options
- Planning migrations or major refactors
- Establishing coding standards or conventions

## Format
See `.cursorrules` in this directory for the template and guidelines.

## Examples
- `2025-01-15-choice-of-queue-system.md`
- `2025-01-20-database-strategy.md`
- `2025-01-25-api-versioning-approach.md`
