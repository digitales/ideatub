# Technical Specifications

This directory contains technical specifications, requirements, and acceptance criteria.

## Purpose
Document the **"what"** that needs to be built - requirements, features, and acceptance criteria.

## When to Add a Spec
- Defining a new feature
- Documenting API contracts
- Specifying data models
- Creating user stories
- Establishing acceptance criteria

## Format
See `.cursorrules` in this directory for the template and guidelines.

## Examples
- `vehicle-valuation-api-spec.md`
- `user-authentication-spec.md`
- `mot-reminder-feature-spec.md`
