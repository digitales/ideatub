# Customer Support Investigations

This directory contains customer support investigation records, troubleshooting notes, and issue resolutions.

## Purpose
Document the **"what happened, why, and how we fixed it"** for customer-facing issues and support cases.

## When to Add an Investigation
- Customer reports a bug or issue
- Production incident occurs
- Data inconsistency is discovered
- Performance degradation affects users
- Billing or account issues
- Integration failures
- User experience problems
- Security incidents

## Format
See `.cursorrules` in this directory for the template and guidelines.

## Examples
- `2025-01-15-valuation-api-timeout.md`
- `CASE-1234-missing-vehicle-data.md`
- `2025-01-20-billing-webhook-failure.md`

## Privacy
- Always anonymize customer data
- Use customer IDs instead of emails/names
- Follow GDPR/data protection guidelines
- Mark sensitive investigations appropriately
